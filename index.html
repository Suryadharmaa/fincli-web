// Get Started

Installation

Multiple ways to install FinCLI. Python 3.11+ is required for all methods.

pipx (Recommended)

Recommended

Isolated Python environment. The fincli command is globally available without polluting your system Python.

pip install pipx
pipx ensurepath
pipx install .
fincli

If published to PyPI:

pipx install fincli

npm (Global)

Global CLI

Common "install once, run anywhere" pattern. The npm wrapper creates a local Python venv during install.

npm install -g @drico2008/fincli
fincli
Note

The npm wrapper still requires Python 3.11+ during installation. The postinstall script creates a .npm-python virtual environment inside the installed package.

Local Development

Dev

Clone the repo and install in editable mode for development.

python -m venv .venv
.venv\Scripts\activate
pip install -e ".[dev]"

Alternative using requirements.txt:

pip install -r requirements.txt

⚙ Post-Install Setup

After installing, launch fincli and save API keys directly from the terminal. No manual file editing needed.

# Save AI provider key
/ai_model key groq <your-api-key>
/ai_model key openrouter <your-api-key>
 
# Save market/news provider key
/news_model key finnhub <your-api-key>
/news_model key twelvedata <your-api-key>
⚠ Security
  • API keys are stored locally at ~/.fincli/secrets.env
  • Keys are never printed in full in the terminal
  • Do not commit .env or secrets.env to version control
  • If a key was ever exposed, revoke and rotate it immediately

Why Textual?

FinCLI needs an interactive terminal dashboard, not a static CLI. Textual provides a reactive widget system for the TUI while Rich handles formatted tables, panels, Markdown rendering, and structured output.

// Data & Config

Local Storage

All data stays on your machine. No cloud sync required.

# Local storage paths
~/.fincli/config.json # Preferences
~/.fincli/fincli.db # SQLite database
~/.fincli/fincli.log # Logs
~/.fincli/secrets.env # API keys (masked)

The SQLite database stores watchlist, portfolio, transactions, journal, session history, and market cache. The two-layer cache (runtime memory + persistent SQLite) reduces API rate-limit pressure and speeds up scanner workflows.

// Help

FAQ & Troubleshooting

▸

What is FinCLI?

FinCLI is a modern financial CLI/TUI terminal for market monitoring, technical analysis, AI-assisted research, portfolio tracking, watchlists, trading journals, and configurable market/news provider workflows. It runs entirely in your terminal using Python's Textual framework for an interactive dashboard experience.
▸

Do I need API keys to use FinCLI?

No. FinCLI works out of the box with yfinance as a free delayed data fallback. API keys are optional — you can add them for better coverage, realtime data, or AI features. Use /setup to see the onboarding guide.
▸

"fincli" is not recognized as a command

Reinstall with pip install -e ., pipx install ., or npm install -g . from the project root. Make sure Python 3.11+ is on your PATH. For pipx, run pipx ensurepath after installation.
▸

The TUI looks cramped or broken

Increase your terminal window size. FinCLI's single-column layout with inline command palette needs a reasonable width. Most modern terminals (Windows Terminal, iTerm2, GNOME Terminal) work well at 120+ columns.
▸

API key is not detected

Use /ai_model key <provider> <key> or /news_model key <provider> <key> to save keys directly. Then verify with /config or /provider key status. Keys are stored at ~/.fincli/secrets.env.
▸

Market data fails for a symbol

Check the symbol format for the active provider. Use /symbol normalize <symbol> to see how FinCLI maps it. Test with /provider test <symbol>. For non-US stocks, use Yahoo exchange suffixes like BBRI.JK, HSBA.L, etc.
▸

Is the data realtime?

yfinance data is usually delayed. API providers (Twelve Data, Finnhub, Alpha Vantage) may provide realtime data depending on subscription plan and exchange entitlement. FinCLI labels data status as realtime, delayed, fallback, or unavailable. Use /provider entitlement to check.
▸

Config is corrupted

Delete ~/.fincli/config.json to return to defaults. API keys in secrets.env are separate and will not be affected. Run /doctor after to verify everything is healthy.
▸

How do I run tests?

Run pytest from the project root. Latest verification: 113 tests passed. For the npm wrapper, use npm run check and npm pack --dry-run.
